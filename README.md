# Terraform Modules for AWS

Reusable Terraform modules for provisioning AWS infrastructure with consistent standards.

## What This Repository Demonstrates

This repository shows how production-focused AWS infrastructure can be organized into reusable modules with versioning, examples, and generated interface docs. It highlights practical Terraform skills such as module design, composability, secure defaults, and standardized developer workflows.

## Quick Start

1. Review prerequisites and local setup steps in [INSTALLATION.md](INSTALLATION.md).
2. Pick a module from the list below (for example `vpc`, `s3/bucket`, or `rds/db`).
3. Reference it from your Terraform stack with a pinned `ref`:

```hcl
module "app_vpc" {
  source = "git@github.com:powise/terraform-modules//aws/vpc?ref=aws-vpc-0.3.3"

  vpc_name   = "example-vpc"
  cidr_block = "10.0.0.0/16"
  subnets    = {}
  tags = {
    team    = "platform"
    service = "networking"
    impact  = "high"
  }
}
```

4. Run your standard Terraform flow:
   - `terraform init`
   - `terraform plan`
   - `terraform apply`

## Repository Map

- `README.md`: Entry point with module index and navigation.
- `INSTALLATION.md`: Tooling and setup for local development and contribution.
- `CONTRIBUTING.md`: Workflow, versioning, and module authoring conventions.
- `<service>/<module>/README.md`: Module purpose, usage example, and autogenerated inputs/outputs.
- `current_module_list.sh`: Keeps the module index in this README up to date.

## How Modules Interact

Modules are designed to be independently reusable, but they are commonly composed in stacks:

- Network baseline: `vpc` + `vpc-endpoint` + `security-groups/security-group`
- Data platform: `rds/db` or `dynamodb/table` with `kms` for encryption
- Edge delivery: `s3/bucket` + `cloudfront/distribution` + `route53/records`

Each module exports outputs that can be wired into other modules in your root Terraform configuration.
